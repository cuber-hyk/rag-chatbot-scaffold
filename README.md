# RAG Chatbot Scaffold

> A Claude Code skill for generating production-ready RAG (Retrieval-Augmented Generation) chatbot projects with enterprise-grade architecture.

[![Claude Code](https://img.shields.io/badge/Claude-Code-Skill-blue)](https://github.com/anthropics/claude-code)

## Overview

This skill generates complete RAG chatbot projects with:

- **Three-Layer Architecture**: API → Service → Repository
- **Multi-Vector Database**: Weaviate, Qdrant, Pinecone with plugin interface
- **Multi-LLM Provider**: OpenAI, Anthropic, SiliconCloud, Azure
- **Document Parsing**: PDF, Markdown, Word, Plain Text via LangChain
- **Document Deduplication**: Smart duplicate detection with conflict handling
- **Session Management**: Redis-based conversation history
- **Network Search**: DuckDuckGo, Tavily integration
- **API Key Authentication**: Simple and secure

## Quick Start

### 1. Configure Your Project

```bash
cp project_config.yaml.example my_project_config.yaml
```

Edit `my_project_config.yaml`:
```yaml
project:
  name: my-chatbot
  description: "My RAG chatbot"
  version: "0.1.0"

vector_db:
  provider: weaviate    # weaviate, qdrant, pinecone
  collection_name: documents

llm:
  provider: openai      # openai, anthropic, siliconcloud, azure
  temperature: 0.7
```

### 2. Generate Project

```bash
python scripts/generate_project.py my_project_config.yaml
```

### 3. Configure Environment

```bash
cd my-chatbot
cp .env.example .env
# Edit .env with your API keys
```

### 4. Install & Run

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

## Features

### Document Deduplication

Smart conflict handling for duplicate uploads:

```bash
# Default: Return conflict info
curl -X POST "http://localhost:8000/api/v1/documents" -F "file=@doc.txt"

# Replace existing
curl -X POST "http://localhost:8000/api/v1/documents?on_conflict=replace" -F "file=@doc.txt"

# Skip if duplicate
curl -X POST "http://localhost:8000/api/v1/documents?on_conflict=skip" -F "file=@doc.txt"
```

**Response Examples:**

- **Filename Conflict**: `文件名 'doc.txt' 已存在但内容不同`
- **Duplicate Content**: `文档 'doc.txt' 的内容已存在（文件名: old.txt）`

## API Endpoints

### Chat
```http
POST /api/v1/chat
{ "query": "What is RAG?" }
→ { "response": "...", "thread_id": "..." }
```

### Documents
```http
POST /api/v1/documents
Form: file=<document>

GET /api/v1/documents
→ { "documents": [...] }

DELETE /api/v1/documents/{id}
→ { "success": true }
```

### Sessions
```http
GET /api/v1/sessions/{thread_id}
→ { "thread_id": "...", "data": {...} }

DELETE /api/v1/sessions/{thread_id}
→ { "success": true }
```
